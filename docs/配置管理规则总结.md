## 🎯 核心规则

### 配置文件管理原则

```
本地开发环境（主配置）
    ↓ scp 同步
CPU服务器（副本）
    ↓ scp 同步
GPU服务器（副本）
```

### 四个核心规则

1. **本地为主** - 本地环境是 config.yaml 的唯一权威来源
2. **单向同步** - 配置只能从本地 → 服务器单向同步
3. **修改后同步** - 修改配置后必须同步到相关服务器
4. **禁止服务器修改** - 禁止直接在服务器上修改配置

## ✅ 正确流程

```bash
# 1. 本地修改配置
vim config.yaml

# 2. 同步到GPU服务器
btcquant train sync-config

# 3. 同步到CPU服务器（如需要）
scp config.yaml cpu_server:/root/workspace/btcquant/

# 4. 启动训练或服务
btcquant train start --gpu
```

## ❌ 禁止操作

1. ❌ 直接在服务器上修改 config.yaml
2. ❌ 从服务器复制 config.yaml 到本地
3. ❌ 提交 config.yaml 到 Git
4. ❌ 在多个地方维护不同版本的配置
5. ❌ 不同步就启动训练或服务

## 📋 常见场景

### 场景1：修改训练参数

```bash
vim config.yaml                    # 修改 predict.training.*
btcquant train sync-config         # 同步到GPU
btcquant train start --gpu         # 启动训练
```

### 场景2：修改邮件配置

```bash
vim config.yaml                    # 修改 notification.email.*
python predict/scripts/test_email.py  # 测试配置
btcquant train sync-config         # 同步到GPU
```

### 场景3：修改标签参数

```bash
vim config.yaml                    # 修改 predict.label.*
scp config.yaml cpu_server:/root/workspace/btcquant/  # 同步到CPU
btcquant data prepare              # 重新生成标签
btcquant train sync-data           # 同步数据到GPU
btcquant train start --gpu         # 启动训练
```

### 场景4：新增配置项

```bash
# 1. 更新示例配置（提交到Git）
vim config.yaml.example

# 2. 更新本地配置
vim config.yaml

# 3. 更新代码（如需要）
vim predict/config.py

# 4. 提交代码
git add config.yaml.example predict/config.py
git commit -m "feat: 添加新配置项"
git push

# 5. 同步到服务器
btcquant train sync-config
```

## 🎓 最佳实践

1. **集中管理** - 所有配置修改在本地完成
2. **及时同步** - 修改后立即同步到服务器
3. **定期备份** - 重要配置定期备份
4. **文档记录** - 重要配置变更记录在文档
5. **测试验证** - 修改后测试配置是否生效

## 💡 为什么这样做？

### 优势

1. **统一管理** - 避免配置不一致
2. **版本控制** - 本地配置可以备份和追踪
3. **安全性** - 避免在服务器上误操作
4. **可追溯** - 所有修改都有记录
5. **易于维护** - 单一真实来源

### 避免的问题

1. ❌ 配置不一致导致训练结果不可复现
2. ❌ 服务器配置被意外修改
3. ❌ 不知道哪个配置是最新的
4. ❌ 配置丢失无法恢复
5. ❌ 多人协作时配置冲突

## 🚀 快速命令

```bash
# 同步配置到GPU
btcquant train sync-config

# 同步配置到CPU
scp config.yaml cpu_server:/root/workspace/btcquant/

# 检查配置差异
diff config.yaml <(ssh gpu_server "cat ~/workspace/btcquant/config.yaml")

# 备份配置
cp config.yaml config.yaml.backup.$(date +%Y%m%d)

# 测试邮件配置
python predict/scripts/test_email.py
```
