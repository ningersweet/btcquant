# 邮件配置更新说明

## 📝 更新内容

将邮件配置从环境变量改为从 `config.yaml` 读取，同时保留环境变量覆盖功能。

## 🔄 主要变更

### 1. 修改的文件

- ✅ `predict/training/post_training.py` - 从 Config 类读取邮件配置
- ✅ `config.yaml.example` - 添加详细的邮件配置说明
- ✅ `README.md` - 更新邮件配置文档
- ✅ `COMMANDS.md` - 添加邮件配置注意事项
- ✅ `PROJECT.md` - 更新邮件配置规范
- ✅ `btcquant` - 移除环境变量检查，改为提示配置文件
- ✅ `predict/scripts/test_email.py` - 新增邮件配置测试脚本

### 2. 配置优先级

```
环境变量 > config.yaml > 默认值
```

这样既方便在配置文件中管理，又保留了环境变量的灵活性。

## 📖 使用方法

### 方式1：在 config.yaml 中配置（推荐）

```yaml
notification:
  email:
    enabled: true
    smtp_server: "smtp.gmail.com"
    smtp_port: 587
    smtp_use_tls: true
    smtp_user: "your_email@gmail.com"
    smtp_password: "your_app_password"
    to_email: "your_email@gmail.com"
```

### 方式2：使用环境变量（覆盖配置文件）

```bash
export SMTP_USER=your_email@gmail.com
export SMTP_PASSWORD=your_app_password
export TO_EMAIL=recipient@example.com
```

### 测试邮件配置

```bash
python predict/scripts/test_email.py
```

## 🎯 优势

1. **统一管理**：所有配置集中在 `config.yaml`
2. **易于维护**：不需要单独管理 `.env.email` 文件
3. **灵活性**：环境变量仍可覆盖配置文件
4. **安全性**：`config.yaml` 已在 `.gitignore` 中
5. **易于测试**：提供测试脚本验证配置

## 📋 常见邮箱配置

### Gmail（推荐）
```yaml
smtp_server: "smtp.gmail.com"
smtp_port: 587
smtp_use_tls: true
```
需要[应用专用密码](https://myaccount.google.com/apppasswords)

### QQ邮箱
```yaml
smtp_server: "smtp.qq.com"
smtp_port: 587
smtp_use_tls: true
```
需要在邮箱设置中获取授权码

### 163邮箱
```yaml
smtp_server: "smtp.163.com"
smtp_port: 465
smtp_use_tls: false
```
需要在邮箱设置中获取授权码

## ✅ 迁移步骤

如果之前使用环境变量配置：

```bash

vim config.yaml

# 2. 添加邮件配置（参考上面的示例）

# 3. 测试配置

# 4. 同步到GPU服务器

# 5. （可选）删除旧的环境变量配置
rm predict/.env.email  # 如果存在
```
