## 1. 标签生成分析

### 1.1 当前实现

**文件**: `predict/src/label_generator.py`

**核心逻辑**:
```python
def generate_labels(df: pd.DataFrame, window: int = 24) -> pd.DataFrame:
    for i in range(n - window):
        current_close = df["close"].iloc[i]
        future_slice = df.iloc[i+1:i+1+window]
        
        max_high = future_slice["high"].max()
        min_low = future_slice["low"].min()
        
        upside = max_high - current_close
        downside = current_close - min_low
        
        # 判断方向
        if upside > downside:
            direction = 1  # LONG
            tp_pct = upside / current_close
            sl_pct = downside / current_close
        else:
            direction = -1  # SHORT
            tp_pct = downside / current_close
            sl_pct = upside / current_close
        
        # 计算盈亏比
        if sl_pct > 0:
            rr = tp_pct / sl_pct * direction
        else:
            rr = 0
```

**生成的标签**:
- `y_rr`: 盈亏比（带方向）
- `y_sl_pct`: 止损百分比
- `y_tp_pct`: 止盈百分比
- `y_direction`: 方向 (1=LONG, -1=SHORT)

---

### 1.2 问题分析

#### 🔴 严重问题1: 标签定义过于简单

**问题**:
```python
if upside > downside:
    direction = 1
else:
    direction = -1
```

**缺陷**:
1. **忽略趋势强度**: 只比较大小，不考虑差异程度
2. **噪音信号多**: 即使 upside 和 downside 很接近也会生成信号
3. **震荡市表现差**: 在横盘时会产生大量错误信号

**示例**:
```
情况1: upside=1.01%, downside=1.00%
结果: direction=1 (LONG)
问题: 差异太小，可能是噪音

情况2: upside=5.00%, downside=0.50%
结果: direction=1 (LONG)
问题: 这才是真正的趋势信号
```

**影响**: 导致方向准确率只有 49-51%（接近随机）

---

#### 🔴 严重问题2: 盈亏比计算不稳定

**问题**:
```python
if sl_pct > 0:
    rr = tp_pct / sl_pct * direction
else:
    rr = 0
```

**缺陷**:
1. **除零风险**: 当 sl_pct 很小时，rr 会爆炸
2. **无上下限**: rr 可以是任意大的值
3. **标签分布极度不平衡**: 大量异常值

**示例**:
```
情况1: tp_pct=2%, sl_pct=0.01%
结果: rr = 200 (异常大)

情况2: tp_pct=1%, sl_pct=0.001%
结果: rr = 1000 (完全不合理)
```

**影响**: MAE_RR = 172-252（误差巨大）

---

#### 🔴 严重问题3: 未来信息泄露风险

**问题**:
```python
max_high = future_slice["high"].max()
min_low = future_slice["low"].min()
```

**潜在风险**:
- 使用未来 24 小时的最高/最低价
- 实际交易中无法提前知道
- 可能导致过拟合

**说明**: 
- 这是标签生成，使用未来数据是合理的
- 但需要确保特征不包含未来信息
- ✅ 当前实现正确（特征只用历史数据）

### 1.4 标签生成优化方案

#### 方案1: 添加趋势强度过滤

```python
def generate_labels_v2(df: pd.DataFrame, window: int = 24) -> pd.DataFrame:
    for i in range(n - window):
        current_close = df["close"].iloc[i]
        future_slice = df.iloc[i+1:i+1+window]
        
        max_high = future_slice["high"].max()
        min_low = future_slice["low"].min()
        
        upside = max_high - current_close
        downside = current_close - min_low
        
        # 计算趋势强度
        trend_strength = abs(upside - downside) / current_close
        
        # 过滤弱趋势
        if trend_strength < 0.005:  # 0.5% 阈值
            direction = 0  # 无信号
            continue
        
        # 判断方向
        if upside > downside:
            direction = 1
            tp_pct = upside / current_close
            sl_pct = downside / current_close
        else:
            direction = -1
            tp_pct = downside / current_close
            sl_pct = upside / current_close
        
        # 约束盈亏比
        rr = np.clip(tp_pct / max(sl_pct, 0.001), -100, 100) * direction
```

**优点**:
- 过滤噪音信号
- 只保留明确趋势
- 提高信号质量

#### 方案3: 多目标标签

```python
def generate_labels_v4(df: pd.DataFrame, window: int = 24) -> pd.DataFrame:
    # 生成多个目标
    targets = {
        'direction': [],      # 方向分类
        'return': [],         # 收益率回归
        'volatility': [],     # 波动率回归
        'sharpe': [],         # 夏普比率
    }
    
    for i in range(n - window):
        future_slice = df.iloc[i+1:i+1+window]
        
        # 方向
        direction = 1 if future_slice['close'].iloc[-1] > df['close'].iloc[i] else -1
        
        # 收益率
        return_pct = (future_slice['close'].iloc[-1] - df['close'].iloc[i]) / df['close'].iloc[i]
        
        # 波动率
        returns = future_slice['close'].pct_change()
        volatility = returns.std()
        
        # 夏普比率
        sharpe = return_pct / volatility if volatility > 0 else 0
        
        targets['direction'].append(direction)
        targets['return'].append(return_pct)
        targets['volatility'].append(volatility)
        targets['sharpe'].append(sharpe)
```

**优点**:
- 多维度预测
- 更全面的信息
- 可以组合使用

#### 方案2: 模型集成

```python
class EnsembleModel:
    def __init__(self):
        self.models = {
            'lgbm': LGBMRegressor(),
            'xgb': XGBRegressor(),
            'cat': CatBoostRegressor(),
        }
        self.weights = None
    
    def fit(self, X_train, y_train, X_val, y_val):
        # 训练所有模型
        predictions = {}
        for name, model in self.models.items():
            model.fit(X_train, y_train)
            predictions[name] = model.predict(X_val)
        
        # 学习最优权重
        self.weights = self.optimize_weights(predictions, y_val)
    
    def predict(self, X):
        predictions = []
        for name, model in self.models.items():
            pred = model.predict(X)
            predictions.append(pred * self.weights[name])
        
        return np.sum(predictions, axis=0)
```

```python
def _compute_metrics(self, y_true, y_pred):
    # 方向准确率
    direction_true = np.sign(y_true[:, 0])
    direction_pred = np.sign(y_pred[:, 0])
    direction_acc = np.mean(direction_true == direction_pred)
    
    # MAE
    mae_rr = np.mean(np.abs(y_true[:, 0] - y_pred[:, 0]))
    mae_sl = np.mean(np.abs(y_true[:, 1] - y_pred[:, 1]))
    mae_tp = np.mean(np.abs(y_true[:, 2] - y_pred[:, 2]))
    
    return {
        "direction_accuracy": direction_acc,
        "mae_rr": mae_rr,
        "mae_sl": mae_sl,
        "mae_tp": mae_tp
    }
```

### 3.3 完整评估方案

```python
class ComprehensiveEvaluator:
    def evaluate(self, y_true, y_pred, prices):
        metrics = {}
        
        # 1. 分类指标
        metrics.update(self.classification_metrics(y_true, y_pred))
        
        # 2. 回归指标
        metrics.update(self.regression_metrics(y_true, y_pred))
        
        # 3. 交易指标
        metrics.update(self.trading_metrics(y_true, y_pred, prices))
        
        # 4. 稳定性指标
        metrics.update(self.stability_metrics(y_true, y_pred))
        
        return metrics
    
    def classification_metrics(self, y_true, y_pred):
        direction_true = np.sign(y_true[:, 0])
        direction_pred = np.sign(y_pred[:, 0])
        
        from sklearn.metrics import (
            accuracy_score, precision_score, recall_score,
            f1_score, confusion_matrix, roc_auc_score
        )
        
        return {
            'accuracy': accuracy_score(direction_true, direction_pred),
            'precision': precision_score(direction_true, direction_pred, average='weighted'),
            'recall': recall_score(direction_true, direction_pred, average='weighted'),
            'f1': f1_score(direction_true, direction_pred, average='weighted'),
            'confusion_matrix': confusion_matrix(direction_true, direction_pred),
        }
    
    def trading_metrics(self, y_true, y_pred, prices):
        # 模拟交易
        returns = []
        for i in range(len(y_pred)):
            direction = np.sign(y_pred[i, 0])
            actual_return = (prices[i+1] - prices[i]) / prices[i]
            trade_return = direction * actual_return
            returns.append(trade_return)
        
        returns = np.array(returns)
        
        # 计算指标
        sharpe = np.mean(returns) / np.std(returns) * np.sqrt(252)
        max_dd = self.calculate_max_drawdown(returns)
        win_rate = np.mean(returns > 0)
        profit_factor = abs(returns[returns > 0].sum() / returns[returns < 0].sum())
        
        return {
            'sharpe_ratio': sharpe,
            'max_drawdown': max_dd,
            'win_rate': win_rate,
            'profit_factor': profit_factor,
            'total_return': returns.sum(),
        }
```
