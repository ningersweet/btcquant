## 1. 系统概述

### 1.1 目标

构建一个**逻辑极简、风控刚性、回测与实盘完全一致**的自动化比特币合约交易系统。

### 1.2 范围

| 范围项 | 说明 |
|--------|------|
| 交易标的 | BTCUSDT 永续合约 |
| 时间粒度 | 1 小时 (1H) K 线 |
| 交易所 | Binance |
| 运行模式 | 回测 / 模拟盘 / 实盘 |
| 部署方式 | Docker 微服务架构 |

### 1.3 约束

| 约束项 | 说明 |
|--------|------|
| 时间刚性 | 开仓、平仓严格锚定整点 K 线周期，预测窗口结束即强制平仓 |
| 单一持仓 | 同一时刻最多持有一个仓位 |
| 固定杠杆 | 20 倍杠杆 |
| 风险控制 | 单笔固定风险金额，最大止损限制 3% |

### 1.4 核心设计理念

- **时间刚性**: 开仓、平仓严格锚定交易所标准 K 线周期
- **点位智能**: 模型直接输出具体的止损价和止盈价
- **架构解耦**: 数据、算法、策略、执行四层分离
- **评估分层**: 严格区分"模型预测能力评估"与"系统交易表现评估"

**标签计算公式**

看涨情况：
```
y_tp_pct = (max_high - close) / close
y_sl_pct = (close - min_low) / close
y_rr = y_tp_pct / y_sl_pct  (正数)
```

看跌情况：
```
y_tp_pct = (close - min_low) / close
y_sl_pct = (max_high - close) / close
y_rr = -y_tp_pct / y_sl_pct  (负数)
```

#### 4.3.4 时间衰减加权

```python
def compute_sample_weights(timestamps: pd.Series, decay_lambda: float = 0.001) -> np.ndarray:
    """
    计算样本权重（时间衰减）
    
    Weight = exp(-λ × ΔDays)
    λ 设为使 2 年前的数据权重约为 0.5
    
    λ = -ln(0.5) / 730 ≈ 0.00095
    """
    latest = timestamps.max()
    delta_days = (latest - timestamps).dt.days
    weights = np.exp(-decay_lambda * delta_days)
    return weights
```

#### 4.4.4 仓位计算

```python
def calculate_position_size(
    risk_amount: float,      # 单笔风险金额 (USDT)
    entry_price: float,      # 入场价格
    sl_price: float,         # 止损价格
    leverage: int = 20       # 杠杆倍数
) -> dict:
    """
    计算仓位大小
    
    公式：
    - 止损距离 = |entry_price - sl_price|
    - 止损百分比 = 止损距离 / entry_price
    - 合约数量 = risk_amount / 止损距离
    - 保证金 = (合约数量 × entry_price) / leverage
    
    Returns:
        {
            'quantity': float,      # 合约数量 (BTC)
            'margin': float,        # 所需保证金 (USDT)
            'notional': float,      # 名义价值 (USDT)
            'sl_distance': float,   # 止损距离 (USDT)
            'sl_pct': float         # 止损百分比
        }
    """
    sl_distance = abs(entry_price - sl_price)
    sl_pct = sl_distance / entry_price
    
    quantity = risk_amount / sl_distance
    notional = quantity * entry_price
    margin = notional / leverage
    
    return {
        'quantity': quantity,
        'margin': margin,
        'notional': notional,
        'sl_distance': sl_distance,
        'sl_pct': sl_pct
    }
```

        """平仓"""

#### 4.5.3 模型评估指标

| 指标 | 计算方法 | 理想目标 |
|------|----------|----------|
| 方向命中率 | 预测方向与实际方向一致的比例 | > 55% |
| 盈亏比实现率 | 实际 RR / 预测 RR 的平均值 | 0.8 ~ 1.2 |
| 止损有效性 | 触及 SL 后未继续大幅亏损的比例 | > 60% |
| 止盈覆盖率 | 价格真正触及 TP 的比例 | 40% ~ 60% |
| 价格偏差度 | 预测价格与实际极值的平均绝对误差 | 越小越好 |

#### 4.5.4 系统评估指标

| 指标 | 计算方法 | 理想目标 |
|------|----------|----------|
| 年化收益率 (CAGR) | 复利年化增长率 | > 20% |
| 最大回撤 | 资金曲线最大下跌幅度 | < 15% |
| 夏普比率 | (年化收益 - 无风险利率) / 波动率 | > 1.5 |
| 盈利因子 | 总盈利 / 总亏损 | > 1.5 |
| 时间止损触发率 | 因时间到期平仓的比例 | < 40% |
| 胜率 | 盈利交易数 / 总交易数 | > 45% |

| 风控项 | 措施 |
|--------|------|
| 单笔风险 | 固定风险金额，不超过账户的 2% |
| 最大止损 | 系统强制限制最大止损 3% |

| 异常检测 | 连续亏损 N 次后暂停交易 |

- SQLite 数据库文件不提交到 Git
- 模型文件不提交到 Git
- 日志文件定期清理

| 服务崩溃 | Docker 自动重启 (restart: always) |
| 数据库损坏 | 从备份恢复，重新同步数据 |
| 模型加载失败 | 回退到上一版本模型 |
| API 连接失败 | 指数退避重试，超时后暂停交易 |

### 10.4 监控指标

| 指标 | 监控方式 |
|------|----------|
| 服务存活 | Docker 健康检查 |
| API 响应时间 | 日志分析 |
| 交易执行 | trades.log 分析 |
| 资金曲线 | 每日评估报表 |
