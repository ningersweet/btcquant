## 问题诊断

### 🔴 严重问题

#### 1. 数据量严重不足
**问题**: 
- 请求 2019-2024 年数据（应有 ~48,000 条）
- 实际只获取到 1,499 条（约 2 个月）
- 导致模型训练样本不足

**影响**:
- 模型无法学习长期模式
- 泛化能力差
- 性能指标不可靠

**原因**:
```python
# predict/api.py 中的 limit 参数限制
limit=1500  # 只获取 1500 条数据
```

**解决方案**:
```python
# 方案1: 分批获取
def fetch_all_data(start_time, end_time, batch_size=1500):
    all_data = []
    current_time = start_time
    while current_time < end_time:
        batch = fetch_klines(current_time, batch_size)
        all_data.extend(batch)
        current_time = batch[-1]['timestamp'] + 1
    return all_data

# 方案2: 使用数据库存储历史数据
# 定期同步完整历史数据到本地数据库
```

#### 2. 方向准确率接近随机 (49-51%)
**问题**:
- 验证集方向准确率: 51.57%
- 最终模型方向准确率: 49.33%
- 仅略高于随机猜测 (50%)

**影响**:
- 模型预测能力弱
- 交易胜率低
- 长期盈利困难

**可能原因**:
1. **数据量不足**: 1,499 条样本太少
2. **特征质量**: 当前特征可能不够有效
3. **标签生成**: 标签定义可能不合理
4. **模型复杂度**: 可能过拟合或欠拟合

---

#### 3. 盈亏比预测误差大 (MAE_RR = 172-252)
**问题**:
- 验证集 MAE_RR: 251.84
- 最终模型 MAE_RR: 172.07
- 误差非常大

**影响**:
- 盈亏比预测不可靠
- 风险管理困难
- 可能导致错误的仓位管理

**原因分析**:
```python
# label_generator.py 中的标签计算
rr = tp_pct / sl_pct * direction

# 问题：
# 1. 当 sl_pct 很小时，rr 会非常大
# 2. 没有对异常值进行处理
# 3. 标签分布可能严重不平衡
```

#### 5. 特征工程问题

**5.1 NaN/Inf 处理简单粗暴**
```python
# features/api.py
result_df = result_df.fillna(0)  # 直接填充为 0
```

**问题**:
- 丢失了"缺失"的信息
- 可能引入错误的特征值
- 影响模型学习

**建议**:
```python
# 更好的处理方式
# 1. 前向填充（适用于价格类特征）
result_df = result_df.fillna(method='ffill')

# 2. 使用中位数/均值填充
result_df = result_df.fillna(result_df.median())

# 3. 保留 NaN，在模型训练时处理
# LightGBM 原生支持 NaN
```

**5.2 特征归一化不一致**
```python
# 有些特征归一化了
result["ema_{period}"] = (ema - df["close"]) / df["close"]

# 有些特征没有归一化
result["obv_norm"] = (obv - obv_min) / obv_range
```

**建议**: 统一归一化策略

#### 6. 标签生成问题

**6.1 标签定义可能不合理**
```python
# 当前逻辑
if upside > downside:
    direction = 1  # LONG
    tp_pct = upside / current_close
    sl_pct = downside / current_close
else:
    direction = -1  # SHORT
    tp_pct = downside / current_close
    sl_pct = upside / current_close
```

**问题**:
- 简单比较 upside 和 downside
- 没有考虑趋势强度
- 没有考虑波动率
- 可能产生大量噪音信号

**建议**:
```python
# 改进方案：考虑趋势强度
trend_strength = abs(upside - downside) / current_close
if trend_strength < threshold:
    # 震荡市，不生成信号
    direction = 0
elif upside > downside:
    direction = 1
else:
    direction = -1
```

**6.2 盈亏比计算问题**
```python
if sl_pct > 0:
    rr = tp_pct / sl_pct * direction
else:
    rr = 0
```

**问题**:
- 当 sl_pct 很小时，rr 会爆炸
- 没有上下限约束
- 导致标签分布极度不平衡

**建议**:
```python
# 添加约束
rr = np.clip(tp_pct / max(sl_pct, 0.001), -100, 100) * direction
```

#### 7. 模型训练问题

**7.1 验证集划分不合理**
```python
# model_trainer.py
X_train, X_val, y_train, y_val = train_test_split(
    X, y, test_size=val_size, shuffle=False
)
```

**问题**:
- 只使用 10% 作为验证集
- 时间序列数据不应该 shuffle=False 后简单切分
- 可能导致数据泄露

**建议**:
```python
# 使用时间序列交叉验证
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
for train_idx, val_idx in tscv.split(X):
    X_train, X_val = X[train_idx], X[val_idx]
    y_train, y_val = y[train_idx], y[val_idx]
    # 训练和评估
```

**7.2 样本权重计算**
```python
# label_generator.py
weights = np.exp(-decay_lambda * delta_days)
```

**问题**:
- 时间衰减可能过于激进
- 没有考虑样本质量
- 可能忽略重要的历史模式
