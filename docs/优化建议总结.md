**解决方案**:
```python
# predict/api.py - 实现分批获取
async def fetch_all_klines(start_time, end_time, batch_size=1500):
    all_data = []
    current_time = start_time
    
    while current_time < end_time:
        batch = await fetch_klines(
            start_time=current_time,
            limit=batch_size
        )
        if not batch:
            break
        
        all_data.extend(batch)
        current_time = batch[-1]['timestamp'] + 3600000  # +1小时
        
        # 避免请求过快
        await asyncio.sleep(0.1)
    
    return all_data
```

**优先级**: 🔥 最高
**预计时间**: 2-4 小时
**预期效果**: 数据量增加 32 倍，模型性能提升 10-20%

### 问题2: 标签生成过于简单 🔴

**现状**:
```python
if upside > downside:
    direction = 1
else:
    direction = -1
```

**问题**:
- 忽略趋势强度
- 产生大量噪音信号
- 盈亏比计算不稳定（MAE_RR = 172-252）

**解决方案**:
```python
def generate_labels_improved(df, window=24):
    for i in range(n - window):
        current_close = df["close"].iloc[i]
        future_slice = df.iloc[i+1:i+1+window]
        
        max_high = future_slice["high"].max()
        min_low = future_slice["low"].min()
        
        upside = max_high - current_close
        downside = current_close - min_low
        
        # 1. 添加趋势强度过滤
        trend_strength = abs(upside - downside) / current_close
        if trend_strength < 0.005:  # 0.5% 阈值
            continue  # 跳过弱趋势
        
        # 2. 判断方向
        if upside > downside:
            direction = 1
            tp_pct = upside / current_close
            sl_pct = downside / current_close
        else:
            direction = -1
            tp_pct = downside / current_close
            sl_pct = upside / current_close
        
        # 3. 约束盈亏比（防止异常值）
        rr = np.clip(
            tp_pct / max(sl_pct, 0.001),  # 防止除零
            -100, 100  # 上下限
        ) * direction
        
        y_rr[i] = rr
        y_sl_pct[i] = sl_pct
        y_tp_pct[i] = tp_pct
```

**优先级**: 🔥 最高
**预计时间**: 2-3 小时
**预期效果**: 
- 方向准确率提升至 55-60%
- MAE_RR 降低至 50-80

### 问题3: 评估指标不全面 🔴

**现状**: 只有 4 个指标
- direction_accuracy: 49.33%
- mae_rr: 172.07
- mae_sl: 0.000635
- mae_tp: 0.001795

**缺失**: 精确率、召回率、夏普比率、最大回撤等

**解决方案**:
```python
def comprehensive_evaluation(y_true, y_pred, prices):
    metrics = {}
    
    # 1. 分类指标
    direction_true = np.sign(y_true[:, 0])
    direction_pred = np.sign(y_pred[:, 0])
    
    metrics['accuracy'] = accuracy_score(direction_true, direction_pred)
    metrics['precision'] = precision_score(direction_true, direction_pred, average='weighted')
    metrics['recall'] = recall_score(direction_true, direction_pred, average='weighted')
    metrics['f1'] = f1_score(direction_true, direction_pred, average='weighted')
    
    # 2. 回归指标
    metrics['rmse_rr'] = np.sqrt(mean_squared_error(y_true[:, 0], y_pred[:, 0]))
    metrics['r2_rr'] = r2_score(y_true[:, 0], y_pred[:, 0])
    
    # 3. 交易指标（模拟交易）
    returns = []
    for i in range(len(y_pred)):
        direction = np.sign(y_pred[i, 0])
        actual_return = (prices[i+1] - prices[i]) / prices[i]
        trade_return = direction * actual_return
        returns.append(trade_return)
    
    returns = np.array(returns)
    metrics['sharpe_ratio'] = np.mean(returns) / np.std(returns) * np.sqrt(252)
    metrics['max_drawdown'] = calculate_max_drawdown(returns)
    metrics['win_rate'] = np.mean(returns > 0)
    metrics['total_return'] = returns.sum()
    
    return metrics
```

**优先级**: 🔥 高
**预计时间**: 3-4 小时
**预期效果**: 全面了解模型性能，发现潜在问题

### 问题4: 模型训练不够稳健 ⚠️

**现状**:
- 单次 90/10 划分
- 只使用 LightGBM
- 验证集太小（150 样本）

**解决方案**:
```python
# 1. 时间序列交叉验证
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
cv_scores = []

for train_idx, val_idx in tscv.split(X):
    X_train, X_val = X[train_idx], X[val_idx]
    y_train, y_val = y[train_idx], y[val_idx]
    
    model.fit(X_train, y_train)
    score = evaluate(model, X_val, y_val)
    cv_scores.append(score)

print(f"CV Score: {np.mean(cv_scores):.4f} ± {np.std(cv_scores):.4f}")

# 2. 模型对比
models = {
    'lgbm': LGBMRegressor(),
    'xgb': XGBRegressor(),
    'catboost': CatBoostRegressor(),
}

for name, model in models.items():
    score = cross_val_score(model, X, y, cv=tscv)
    print(f"{name}: {score.mean():.4f}")

# 3. 模型集成
ensemble = VotingRegressor([
    ('lgbm', LGBMRegressor()),
    ('xgb', XGBRegressor()),
    ('cat', CatBoostRegressor()),
])
```

**优先级**: 🟡 中
**预计时间**: 4-6 小时
**预期效果**: 模型更稳健，性能提升 5-10%

### 问题5: 特征工程可以改进 ⚠️

**现状**:
- 39 个特征
- NaN 直接填充为 0
- 缺少收益率、K线形态、波动率特征

**解决方案**:
```python
# 1. 改进 NaN 处理
result_df = result_df.fillna(method='ffill')  # 前向填充
# 或保留 NaN，LightGBM 原生支持

# 2. 添加缺失特征
# 收益率
for period in [1, 3, 5, 10, 20]:
    df[f'return_{period}'] = df['close'].pct_change(periods=period)

# K线形态
df['body_ratio'] = abs(df['close'] - df['open']) / (df['high'] - df['low'])
df['upper_shadow'] = (df['high'] - df[['open', 'close']].max(axis=1)) / (df['high'] - df['low'])
df['lower_shadow'] = (df[['open', 'close']].min(axis=1) - df['low']) / (df['high'] - df['low'])

# 波动率
returns = df['close'].pct_change()
for period in [5, 10, 20]:
    df[f'volatility_{period}'] = returns.rolling(window=period).std()

# 3. 特征选择
from sklearn.feature_selection import SelectKBest, f_regression
selector = SelectKBest(f_regression, k=30)
X_selected = selector.fit_transform(X, y)
```

**优先级**: 🟡 中
**预计时间**: 4-6 小时
**预期效果**: 特征质量提升，模型性能提升 3-5%
