# 超参数优化指南

## 📋 概述

本系统使用 **Optuna** 框架进行自动超参数优化，通过贝叶斯优化算法（TPE Sampler）自动搜索最佳模型参数组合。

## 📊 优化目标

### 综合评分公式

```
Score = 0.7 × 验证准确率 + 0.3 × (1 - 验证损失)
```

- **主要目标**：最大化验证集准确率
- **次要目标**：最小化验证集损失
- **早停条件**：验证损失连续N个epoch不下降

### 评估指标

1. **方向准确率** (Direction Accuracy)
   - 预测符号与真实符号一致的比例
   - 目标：> 55%

2. **MAE_RR** (Mean Absolute Error - Risk Reward)
   - 盈亏比预测的平均绝对误差
   - 目标：< 50

3. **MAE_SL** (Mean Absolute Error - Stop Loss)
   - 止损百分比预测的平均绝对误差
   - 目标：< 0.005 (0.5%)

4. **MAE_TP** (Mean Absolute Error - Take Profit)
   - 止盈百分比预测的平均绝对误差
   - 目标：< 0.01 (1%)

## 🔍 结果解读

### 1. 优化结果 (optimization)

- **best_params**: 找到的最佳超参数组合
- **best_score**: 最佳综合得分（0-1，越高越好）
- **best_metrics**: 最佳试验的详细指标
- **n_trials**: 实际执行的试验次数
- **best_trial_number**: 最佳试验的编号

### 2. 最终模型指标 (final_metrics)

使用最佳参数在全量数据（训练集+验证集）上训练后的性能指标。

### 3. 性能基准

| 指标 | 及格 | 良好 | 优秀 |
|------|------|------|------|
| 方向准确率 | > 52% | > 55% | > 60% |
| MAE_RR | < 80 | < 60 | < 40 |
| MAE_SL | < 0.008 | < 0.005 | < 0.003 |
| MAE_TP | < 0.015 | < 0.010 | < 0.007 |

## 💡 优化建议

### 1. 数据量影响

- **小数据集**（< 5000 条）：可能过拟合，建议减少 `n_estimators` 和 `max_depth`
- **大数据集**（> 20000 条）：可以使用更复杂的模型，增加试验次数

### 2. 时间预算

- **快速迭代**：10-20 次试验，10-30 分钟
- **标准优化**：50 次试验，30-60 分钟
- **精细调优**：100+ 次试验，2-4 小时

### 3. 优化策略

1. **第一轮**：使用较少试验次数（10-20）快速探索
2. **第二轮**：在第一轮基础上，缩小参数范围，增加试验次数（50-100）
3. **第三轮**：针对关键参数进行精细调优

### 4. 监控指标

优化过程中重点关注：
- 方向准确率是否稳定提升
- MAE 是否持续下降
- 是否出现过拟合（训练集表现好但验证集差）

## ⚠️ 注意事项

1. **资源消耗**
   - 优化过程会占用较多 CPU 和内存
   - 建议在低峰期执行
   - 可以通过 `timeout` 参数限制最大运行时间

2. **数据质量**
   - 确保数据已同步完整
   - 检查是否有缺失值或异常值
   - 验证特征计算是否正确

3. **模型保存**
   - 优化后的模型会自动保存
   - 模型 ID 格式：`model_optimized_YYYYMMDD_HHMMSS`
   - 建议记录每次优化的参数和结果

4. **版本管理**
   - 保留多个优化版本进行对比
   - 在测试集上验证性能
   - 逐步替换生产模型

## 🛠️ 故障排查

### 1. 优化失败

**问题**：返回 500 错误
**解决**：
- 检查数据是否充足
- 查看服务日志：`docker-compose logs predict-service`
- 验证时间范围是否正确

### 2. 优化时间过长

**问题**：超过预期时间
**解决**：
- 减少 `n_trials` 数量
- 设置合理的 `timeout`
- 检查服务器资源使用情况

### 3. 性能未提升

**问题**：优化后性能反而下降
**解决**：
- 检查是否过拟合
- 增加验证集大小
- 调整优化目标权重
- 尝试更多试验次数

1. **渐进式优化**：从小规模开始，逐步扩大
2. **记录实验**：保存每次优化的配置和结果
