📄 项目需求文档：轻量级加密货币时序预测模型 (CPU 优化版)

1. 项目背景与目标
构建一个基于 PyTorch 的深度学习模型，用于加密货币 K 线数据的多任务预测（方向分类 + 价格回归）。
*   核心约束：必须在低算力 CPU 环境（服务器 2 核，本地 6 核）上高效运行。严禁使用 Transformer/Attention 机制。
*   架构策略：采用 “本地全量预训练 + 服务器增量微调” 的部署模式。
*   模型类型：浅层多尺度 1D-CNN + MLP。

2. 技术栈要求
*   语言: Python 3.9+
*   框架: PyTorch (CPU 后端)
*   数据处理: Pandas, NumPy
*   依赖库: torch, pandas, numpy, scikit-learn (仅用于数据划分，不用于建模)
*   禁止项: 禁止引入 transformers 库，禁止使用复杂的 GPU 专用算子，禁止使用多进程数据加载 (num_workers > 0)。

3. 模型架构规范 (Model Architecture)

请生成一个名为 LightWeightCPUNet 的 nn.Module 类，需满足以下细节：

3.1 输入输出定义
*   输入: (Batch_Size, Sequence_Length, Features)
    *   Sequence_Length: 固定为 288 (代表 24 小时，5 分钟一根 K 线)。
    *   Features: 固定为 5 (Open, High, Low, Close, Volume)。
*   输出 (多任务头):
    1.  分类头: (Batch_Size, 3) -> 对应 [Hold, Long, Short] 的概率分布。
    2.  回归头: (Batch_Size, 3) -> 对应 [Entry_Offset, Take_Profit_Distance, Stop_Loss_Distance]。

3.2 网络结构设计
1.  特征提取层 (多尺度并行卷积):
    *   将输入从 (B, T, F) 转置为 (B, F, T) 以适配 Conv1d。
    *   设计 3 个并行分支，每个分支包含 Conv1d -> BatchNorm1d -> GELU：
        *   短期分支: kernel_size=3, padding=1 (捕捉瞬间波动)。
        *   中期分支: kernel_size=12, padding=6 (捕捉小时级趋势)。
        *   长期分支: kernel_size=48, padding=24 (捕捉日线级背景)。
    *   所有分支的 out_channels 设为 16 (保持极轻量)。
2.  池化层:
    *   对每个分支输出使用 AdaptiveAvgPool1d(1) 进行全局平均池化，将时间维压缩为 1。
3.  融合层:
    *   拼接三个分支的输出 (维度: 16 times 3 = 48)。
    *   通过一个全连接层 Linear(48, 32) + Dropout(0.2) + GELU。
4.  输出头:
    *   head_cls: Linear(32, 3)
    *   head_reg: Linear(32, 3)

3.3 参数量控制
*   总参数量应控制在 20,000 (20k) 以内，确保能完全载入 CPU 缓存。
