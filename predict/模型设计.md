比特币量化预测模型实现
1. 角色设定
你是一位资深量化算法工程师和Python 专家。你的任务是实现一个用于比特币日内合约交易的多输出回归预测模型。该模型将作为量化交易系统的核心“大脑”，负责预测未来 1 小时内的盈亏比 (RR)、建议止损价 (SL) 和建议止盈价 (TP)。
2. 项目背景与目标
交易标的: BTCUSDT 永续合约。
时间粒度: 1 小时 (1H) K 线。
核心策略: 基于固定时间网格（整点）触发，模型输出具体点位，配合刚性时间止损。
数据策略: 使用 2019 年至今的全量数据，采用时间衰减加权进行训练，每月重训一次。
技术栈: pandas, numpy, pandas-ta, scikit-learn, lightgbm, optuna (可选), joblib。
3. 详细需求规格

结构特征: 过去 N 周期的高低点 (Highest High, Lowest Low)，距离最近整数关口的距离，小时/星期特征。
归一化: 所有价格相关特征需转化为相对比率或百分比，避免绝对价格影响。

3.3 推理与服务模块 (model_inference.py)
输入: 最新的市场数据 DataFrame (包含必要的历史窗口以计算特征)。
功能:
加载训练好的模型和预处理配置。
实时计算特征 (必须与训练时逻辑完全一致)。
模型预测，输出 [rr_score, sl_pct, tp_pct]。
后处理校验 (Logic Gate):
将百分比还原为绝对价格。
校验 1: 若 rr_score > 0，必须满足 SL < Current < TP。
校验 2: 重新计算实际 RR，若 < 1.5 则视为无效信号。
校验 3: 过滤极端止损 (如 > 5% 或 < 0.1%)。
输出: 返回标准化的 JSON 对象 {valid: bool, signal: "LONG"/"SHORT"/"NONE", rr: float, sl_price: float, tp_price: float}。
3.4 主流程脚本 (main_pipeline.py)
提供一个入口脚本，支持两种模式：
--mode train: 执行全量数据加载、特征工程、加权、训练、评估、保存。
--mode infer: 读取最新数据，运行推理，打印预测结果。
