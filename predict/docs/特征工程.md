在模型训练过程中，特征工程（Feature Engineering）是决定模型上限的关键环节。原始数据（OHLCV）直接输入往往包含大量噪音且量纲不一，模型难以收敛。

基于 TCN 架构 和 2核CPU 的限制，我们需要构建一套计算轻量、信息密度高、无未来函数的特征集。

以下是完整的特征清单及计算公式，分为 基础标准化特征、技术指标特征 和 时序结构特征 三类。

📊 完整特征列表 (Total: 18 Features)

我们将输入维度从原始的 5 维扩展到 18 维。所有特征必须在 t 时刻完全可见（严禁使用未来数据）。

基础标准化特征 (Base Normalized Features) - 5维目的：消除价格绝对值影响，让模型关注相对变化。
索引   特征名称   符号   计算公式   说明
0   对数收益率   f_{ret}   ln(frac{Close_t}{Close_{t-1}})   替代原始收盘价，近似正态分布，更稳定

1   高位偏移率   f_{high}   frac{High_t - Close_{t-1}}{Close_{t-1}}   衡量向上插针的幅度

2   低位偏移率   f_{low}   frac{Low_t - Close_{t-1}}{Close_{t-1}}   衡量向下插针的幅度

3   实体比例   f_{body}   frac{Close_t - Open_t}{Close_{t-1}}   衡量K线多空力量强弱 (正为阳，负为阴)

4   相对成交量   f_{vol}   ln(frac{Volume_t}{text{SMA}(Volume, 20)_t} + epsilon)   当前成交量相对于过去20根均量的倍数取对数

注意：原始方案中的 Z-Score 标准化是在整个输入窗口 (288) 上进行的预处理，而这里的特征是逐根K线计算的微观特征。两者结合效果最佳：先计算微观特征，再对微观特征序列做滚动 Z-Score。

趋势与动量特征 (Trend & Momentum) - 6维目的：帮助 TCN 快速捕捉当前市场方向，减少卷积层学习基础趋势的压力。
索引   特征名称   符号   计算公式   参数建议
5   RSI 相对强弱   f_{rsi}   100 - frac{100}{1 + RS}  RS = frac{text{AvgGain}}{text{AvgLoss}}   窗口 N=14

6   MACD 差离值   f_{macd}   EMA(Close, 12) - EMA(Close, 26)   标准参数 (12, 26)

7   MACD 信号线   f_{macdsig}   EMA(f{macd}, 9)   标准参数 (9)

8   布林带带宽   f_{bbw}   frac{UpperBand - LowerBand}{MiddleBand}   窗口 N=20, sigma=2 衡量波动率压缩/扩张

9   布林带位置   f_{bbp}   frac{Close_t - LowerBand}{UpperBand - LowerBand}   范围 [0, 1]，衡量价格在通道中的相对位置

10   线性回归斜率   f_{slope}   text{Slope}(text{LinearReg}(Close, 20))   窗口 N=20 量化短期趋势角度

波动率与极值结构特征 (Volatility & Structure) - 7维目的：这是本策略的核心。直接告诉模型“现在的价格距离近期极值有多远”，辅助“极值时序”逻辑。
索引   特征名称   符号   计算公式   物理含义
11   ATR 归一化波动   f_{atr}   frac{ATR(14)_t}{Close_t}   当前平均真实波幅占价格比例

12   近高距离率   f_{dist_h}   frac{Close_t - max(High_{t-24 dots t})}{max(High_{t-24 dots t})}   当前价距离过去24小时最高点的差距 (负值)

13   近低距离率   f_{dist_l}   frac{Close_t - min(Low_{t-24 dots t})}{min(Low_{t-24 dots t})}   当前价距离过去24小时最低点的差距 (正值)

14   上下影线比   f_{wick_ratio}   frac{(High_t - Close_t) + (Open_t - Low_t)}{ Close_t - Open_t  + epsilon}   衡量拒绝程度 (值越大，变盘概率越高)

15   成交量突变   f_{vol_spike}   frac{Volume_t}{max(Volume_{t-12 dots t-1}) + epsilon}   当前量是否是过去1小时的峰值

16   高低点时序标记   f_{seqflag}   text{OneHot}(text{Pattern}{t})   关键特征：0: 无明确极值1: 刚创24h新低2: 刚创24h新高捕捉亚盘/欧盘/美盘的周期性规律

注：特征 16 和 17 使得输入总维度变为 18 (如果时间嵌入算2维) 或 19。为了对齐矩阵，建议固定为 18 维：
将 f_{time} 合并为 1 维：sin(dots) 即可，或者去掉 f_{vol_spike}。
推荐配置：保留正弦和余弦 (2维)，去掉 f_{macd_sig} (因为 MACD 差离值已包含主要信息)，保持总数 18 维。

🛠️ 特征计算代码实现 (Python/Pandas)

为了确保训练和实盘的一致性，请使用以下标准化函数生成特征矩阵：

import pandas as pd
import numpy as np
import talib # 建议使用 TA-Lib 库，速度极快

def generate_features(df):
    """
    输入: df 包含 ['open', 'high', 'low', 'close', 'volume', 'timestamp']
    输出: numpy array (N, 18)
    """
    df = df.copy()
    
    # --- 1. 基础特征 ---
    df['ret'] = np.log(df['close'] / df['close'].shift(1))
    df['high_off'] = (df['high'] - df['close'].shift(1)) / df['close'].shift(1)
    df['low_off'] = (df['low'] - df['close'].shift(1)) / df['close'].shift(1)
    df['body'] = (df['close'] - df['open']) / df['close'].shift(1)
    df['vol_rel'] = np.log(df['volume'] / df['volume'].rolling(20).mean() + 1e-8)
    
    # --- 2. 趋势特征 ---
    df['rsi'] = talib.RSI(df['close'], timeperiod=14) / 100.0 # 归一化到 0-1
    df['macd'], df['macd_sig'], _ = talib.MACD(df['close'], 12, 26, 9)
    df['macd'] = df['macd'] / df['close'] # 归一化
    # df['macd_sig'] 被移除以保持维度平衡
    
    upper, middle, lower = talib.BBANDS(df['close'], timeperiod=20, nbdevup=2, nbdevdn=2)
    df['bbw'] = (upper - lower) / middle
    df['bbp'] = (df['close'] - lower) / (upper - lower + 1e-8)
    
    # 线性回归斜率 (使用 scipy 或手动计算，这里用简单近似)
    df['slope'] = talib.LINEARREG_SLOPE(df['close'], timeperiod=20) / df['close']
    
    # --- 3. 波动与结构特征 ---
    df['atr'] = talib.ATR(df['high'], df['low'], df['close'], timeperiod=14) / df['close']
    
    # 24小时 (288根) 极值距离
    roll_high = df['high'].rolling(288).max()
    roll_low = df['low'].rolling(288).min()
    df['dist_h'] = (df['close'] - roll_high) / (roll_high + 1e-8)
    df['dist_l'] = (df['close'] - roll_low) / (roll_low + 1e-8)
    
    # 影线比
    upper_wick = df['high'] - np.maximum(df['close'], df['open'])
    lower_wick = np.minimum(df['close'], df['open']) - df['low']
    body_abs = np.abs(df['close'] - df['open'])
    df['wick_ratio'] = (upper_wick + lower_wick) / (body_abs + 1e-8)
    
    # 成交量突变 (过去12根)
    roll_vol_max = df['volume'].rolling(12).max().shift(1) # shift避免包含自身
    df['vol_spike'] = df['volume'] / (roll_vol_max + 1e-8)
    
    # 高低点时序标记 (简化版：是否为窗口极值)
    df['is_new_high'] = (df['high'] == roll_high).astype(float)
    df['is_new_low'] = (df['low'] == roll_low).astype(float)
    df['seq_flag'] = df['is_new_high'] - df['is_new_low'] # 1:新高, -1:新低, 0:其他
    
    # 时间嵌入 (小时)
    df['hour'] = pd.to_datetime(df['timestamp']).dt.hour
    df['time_sin'] = np.sin(2 * np.pi * df['hour'] / 24)
    # df['time_cos'] 移除以保持 18 维，或替换掉 vol_spike
    
    # --- 选择最终 18 列 ---
    feature_cols = [
        'ret', 'high_off', 'low_off', 'body', 'vol_rel',
        'rsi', 'macd', 'bbw', 'bbp', 'slope',
        'atr', 'dist_h', 'dist_l', 'wick_ratio', 'vol_spike',
        'seq_flag', 'time_sin', 'close' # 最后一个用 close 的 log 填充或重复 ret，建议再加一个 'vol_ma_ratio'
    ]
    
    # 修正：确保正好 18 个有意义的特征
    # 重新整理列表：
    final_cols = [
        'ret', 'high_off', 'low_off', 'body', 'vol_rel', # 5
        'rsi', 'macd', 'bbw', 'bbp', 'slope',            # 5
        'atr', 'dist_h', 'dist_l', 'wick_ratio', 'vol_spike', # 5
        'seq_flag', 'time_sin', 'macd_sig'                # 3 (把 macd_sig 加回来，去掉 vol_spike 也可以，这里保留 macd_sig)
    ]
    
    # 处理 NaN (填充或丢弃)
    df_feat = df[final_cols].fillna(0)
    
    return df_feat.values

后续步骤：对生成的特征矩阵进行 Rolling Z-Score 标准化
def rolling_zscore(data, window=288):
    # data shape: (N, 18)
    df = pd.DataFrame(data)
    mean = df.rolling(window=window).mean()
    std = df.rolling(window=window).std()
    return ((df - mean) / (std + 1e-8)).values

💡 关键注意事项

NaN 处理：
    由于使用了 rolling(288) 和 shift，前 288 根数据会产生 NaN。
    训练时：直接丢弃这些样本（Label 构造时也会因为缺乏历史数据而无法生成有效标签，天然对齐）。
    实盘时：必须积累满 288 根 K 线后才开始推理。

一致性原则：
    训练集、验证集和实盘推理必须使用完全相同的特征计算代码。
    任何参数（如 RSI 的 14，BB 的 20）的修改都必须重新训练模型。

计算开销：
    上述所有特征均为 O(N) 复杂度，在 2 核 CPU 上计算 288 根数据的特征耗时 < 5 毫秒，完全不会成为瓶颈。

为什么没有 Order Book 数据？
    订单簿数据量太大且噪音极高，会显著增加 2 核 CPU 的处理负担。本策略基于 K 线极值逻辑，OHLCV 衍生特征已足够捕捉“极值时序”模式。

这套 18 维特征体系 专为 TCN 设计，既包含了趋势、波动率等统计信息，又显式编码了策略核心的“极值距离”和“时序标记”，能最大程度发挥模型的预测能力。
